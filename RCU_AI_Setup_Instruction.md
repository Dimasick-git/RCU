# 🤖 Универсальный AI-Автоисправитель для RCU

Я значительно улучшил систему. Теперь это не просто "фиксер ошибок билда", а полноценный AI-ассистент, который может чинить **любые** ошибки в репозитории и выполнять ваши команды.

## 🚀 Основные возможности:
1.  **Авто-фикс ошибок билда**: Если основной workflow (`Build Horizon OC Zeus`) падает, AI автоматически анализирует логи и присылает PR с исправлением.
2.  **Команда `/fix`**: Теперь вы можете написать `/fix` в комментариях к любому Issue или Pull Request. AI прочитает описание проблемы и предложит код для её решения.
3.  **Универсальность**: Система умеет работать с кодом, конфигурациями и GitHub Actions одновременно.
4.  **Бесплатные модели**: По умолчанию используются лучшие бесплатные модели через OpenRouter (`openrouter/free`).

## ⚙️ Настройка (Секреты GitHub):

Если вы ещё этого не сделали, добавьте эти секреты в **Settings -> Secrets and variables -> Actions**:

| Секрет | Описание |
| :--- | :--- |
| `OPENROUTER_API_KEY` | Ваш ключ от [OpenRouter](https://openrouter.ai/keys). Нужен для работы AI. |
| `GH_PAT` | **Обязательно для PR**: Ваш Personal Access Token (с правами `repo` и `workflow`). Без него AI не сможет создавать ветки и открывать PR. |

## 🛠 Как пользоваться:

### 1. Автоматически
Просто делайте пуши. Если билд сломается, через пару минут в разделе **Pull Requests** появится предложение от AI по исправлению.

### 2. Вручную (Slash Command)
Если у вас есть открытый Issue или PR, где что-то не работает или нужно добавить фичу:
1.  Напишите комментарий `/fix`.
2.  AI проанализирует контекст обсуждения и файлы проекта.
3.  AI создаст PR с реализацией или исправлением.

## 🛡 Безопасность и ограничения
- AI проинструктирован **не трогать** и не ломать критические компоненты: `Ryazha-Авто` и `VRR`.
- Все изменения приходят в виде **Pull Request**, поэтому вы всегда можете проверить код перед тем, как принять его.
